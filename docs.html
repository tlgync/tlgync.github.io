Documentation

Everything you need to run AI coding agents inside mrd — from your first launch to the settings most people never have to touch.

01 Overview

mrd is a native macOS terminal built around AI coding agents. Instead of juggling agents across scattered terminal tabs, you give each project a workspace, open Claude Code or Codex inside it, and let mrd keep track of what every agent is doing — even while you're looking at something else.

Three things make it different from a plain terminal:

  • It watches your agents. A small heads-up display under the notch shows every live agent and whether it's running, finished, or waiting on you.
  • It talks back. When an agent needs a decision, you answer right there — no window-switching.
  • It remembers. Quit and reopen, and your layout plus each agent's conversation come back.

Under the hood it's one process — no background daemon, no Electron. The terminal itself is rendered by libghostty, the engine behind the Ghostty terminal, so it's fast and reuses your existing Ghostty configuration.

ℹ️

mrd only offers an agent in its + menu if it can actually find that CLI on your shell's PATH. If Open Claude or Open Codex is missing, see Troubleshooting.

03 Install & first launch

  1. Download the latest build from the releases page (a .zip).
  2. Unzip it (double-click), then drag mrd.app into your Applications folder.
  3. Open it. Builds are signed and notarized by Apple, so it launches with no Gatekeeper warning and no Terminal step.

On first launch, if you have Claude Code or Codex installed, mrd quietly sets up the notification hooks those agents need (see Notifications). There's nothing to configure.

⚠️

If macOS ever says the app is “damaged”, you're almost certainly opening an older, un-notarized build. Grab the newest one — current releases are notarized. As a one-time fix for an old build: xattr -dr com.apple.quarantine /Applications/mrd.app

04 Staying up to date

mrd updates itself — there's no separate updater app and no daemon.

  • Shortly after launch (and every few hours) it checks for a newer build.
  • If one exists, a small “Update available” footer appears at the bottom of the sidebar. It's invisible when you're already current.
  • Click it to download. mrd verifies the download's checksum, swaps itself in place, and relaunches.

You can also check on demand from the menu bar: mrd → Check for Updates…

ℹ️

If you're running mrd from a read-only spot (like straight out of a mounted disk image), it can't replace itself — it'll open the releases page instead. Move it to Applications for seamless updates. To turn auto-checks off entirely, see Settings.

05 The three panels

The window is split into three regions, the same arrangement you know from Mail or Notes:

PanelWhat lives there
Sidebar (left)Your workspaces and groups. Each row shows the project's branch and folder, plus an attention dot when an agent needs you.
Content (center)The selected workspace: a Safari-style tab band over your terminals, splits, and browser panes.
Inspector (right)Details about the current workspace — listening ports, branch, and more. Hidden by default.

Toggle the side panels with ⌃⌘S (Sidebar) and ⌥⌘I (Inspector). The sidebar and inspector are real macOS glass; your terminal sits on a calm solid background so text stays crisp.

06 Workspaces & groups

A workspace is a project's home — its working directory, its terminals, its agents. A group is a labeled collection of workspaces (e.g. “API”, “Frontend”).

  • New Workspace — ⌘N, or the + at the top of a group.
  • New Group — ⇧⌘N.
  • Jump to a workspace — ⌘1 … ⌘9 selects the Nth one in sidebar order.
  • Reorder — drag rows within or between groups.
  • Close — ⌘W closes the current workspace.

Each row shows the git branch and an abbreviated folder path, so you can tell projects apart at a glance. When an agent finishes or needs input while you're elsewhere, that row lights up — see Notifications & attention.

07 Terminals, tabs & splits

Inside a workspace you get a full terminal, rendered by libghostty.

Tabs

Open a new terminal tab with ⌘T. Tabs sit in a band across the top of the content area; click to switch, or use the + at the end of the band for the new-session menu.

Splits

Divide a pane to see two things at once:

Split right⌘D
Split down⇧⌘D
Focus a neighbor⌥⌘ ← → ↑ ↓
Close pane⇧⌘W

Your Ghostty config comes along

mrd reads your existing Ghostty configuration — ~/.config/ghostty/config and the usual Application Support locations — so your fonts, colors, and keybindings are already in place. You don't keep a second config just for mrd.

ℹ️

By default mrd renders exactly like native Ghostty. It applies no visual overrides of its own unless you explicitly opt in (see MRD_RENDER_OVERRIDES under Settings).

08 Browser panes

Sometimes you need a quick browser next to your terminal — a localhost preview, docs, a PR. mrd can open a browser tab right inside a workspace with ⌥⌘B. From the Browser menu you can also clear browsing history or import data. The browser shares the same tab band as your terminals, so it lives wherever your work does.

09 The Inspector

The Inspector (⌥⌘I) is a context panel for the selected workspace. Its most useful section is Ports: the ports your project is currently listening on, detected live — handy for spotting which dev server is up without hunting through logs. Branch and other project details live here too, keeping the sidebar rows clean.

10 Running Claude & Codex

Open an agent from the + menu — you'll find it in the toolbar, at the end of the tab band, on each sidebar group, and on a sidebar right-click. They all open the same menu:

ItemWhat it does
New TerminalA plain shell. Always available.
Open ClaudeStarts a Claude Code session in a fresh pane. Shown only if the claude CLI is on your PATH.
Open CodexStarts a Codex session. Shown only if the codex CLI is on your PATH.

You can also just type claude or codex into any terminal — mrd notices an agent running in a shell and starts tracking it, whether you launched it from the menu or by hand.

How mrd knows what an agent is doing

mrd never reads your screen to guess. It listens to each agent's own hook events — the agent tells mrd when a turn starts, when a tool runs, and when it finishes. That's why the status is reliable: it's the agent's own truth, not an interpretation of terminal output.

11 The notch HUD

The heads-up display is a slim panel pinned under your MacBook's notch (or top-center, on a Mac without one). Collapsed, it shows which agents are alive. Hover — or let an agent notify you — and it expands into a card listing every agent workspace.

  • Live status per agent — green while a turn is running, amber the moment it's waiting on you, quiet grey when idle.
  • Click a row to jump straight to that pane.
  • It never steals focus. The panel can't become the active window, so your typing and terminal focus are never disturbed.

A notification banner slides in when an agent finishes off-screen, gated exactly like the sidebar's attention dot — so you're told once, and it clears when you look.

ℹ️

Prefer the menu bar, or no HUD at all? The agent HUD is configurable — see Settings & opt-outs.

12 Notifications & attention

When an agent finishes a turn or asks for something while you're not looking, mrd raises attention in three coordinated places:

  • A dot and a one-line status on the workspace's sidebar row.
  • A banner in the notch HUD.
  • A native macOS notification, if the window isn't visible.

Attention is smart about clearing itself: it goes away the instant you focus that workspace, the moment the agent starts a new turn, or as soon as the agent resumes working after a pause. You won't get stuck with a “needs you” that lingers after you've already dealt with it.

Hooks set themselves up

To receive these events, mrd installs small hooks into your agents' config on first launch — ~/.claude/settings.json for Claude Code and ~/.codex/hooks.json for Codex. It only writes when something's missing or out of date, and repairs itself if you move the app.

If you ever want to manage them by hand, the bundled CLI can:

# install / refresh hooks for installed agents
/Applications/mrd.app/Contents/Resources/mrd-cli hooks setup

# remove them again
/Applications/mrd.app/Contents/Resources/mrd-cli hooks remove
⚠️

Codex users: Codex ignores hooks it hasn't been told to trust — silently. The first time mrd adds new hooks, Codex will treat them as untrusted until you approve them once, through Codex's own hook-review prompt or its /hooks command. Already-trusted hooks keep working; this only affects newly added ones.

13 Answering questions in place

Agents sometimes stop and ask: a multiple-choice question, or permission to run a tool or edit a file. When that happens and you're not already looking at the pane, mrd surfaces it as a card in the notch HUD — a tidy set of options, or an Allow / Deny prompt. Your choice flows straight back to the agent, which carries on. No window-switching, no hunting for the right tab.

  • Smart routing. If you're already focused on the agent's pane, mrd steps aside and lets the agent's own in-terminal prompt handle it — no duplicate card.
  • Safe by default. If anything goes wrong — the pane closes, mrd quits, the panel is dismissed — the question is simply handed back to the agent's own interface. mrd never auto-denies on your behalf.

This works for both Claude Code and Codex. (Claude additionally supports rich multiple-choice questions; Codex surfaces permission prompts.)

14 Session resume

Close mrd and reopen it — your workspaces, tabs, and splits come back, and so do your agents. mrd restores each agent's pane by replaying its scrollback and resuming the conversation with the agent's native resume command, so you continue the same thread rather than starting over.

When you closed it…On relaunch
An agent with an in-progress conversationResumes that conversation (claude --resume / codex resume).
An agent open but never promptedReopens a fresh agent, ready to go.
You deliberately logged out / exited the agentNothing is reopened — your exit is respected.
ℹ️

Codex doesn't emit a “session ended” signal, so a Codex session you closed on purpose may still reappear after a relaunch. That's a Codex limitation, not a bug in mrd. You can turn resume off entirely in Settings.

15 Voice dictation

You can talk to your agent instead of typing. A microphone button appears in the toolbar whenever the focused pane is a Claude or Codex session — it's deliberately hidden for plain terminals, where free-form dictation rarely makes sense.

How it works

  1. Click the mic to start. The first time, macOS asks for Microphone and Speech Recognition access. mrd locks onto the agent pane you're on and prepares the on-device model.
  2. Speak. A soft glow breathes around the target pane while it's listening (orange for Claude, bright grey for Codex). It keeps listening — it isn't push-to-talk and won't cut out on a pause.
  3. Click the mic again to finish. The transcribed text is pasted into the agent's prompt. mrd does not press Enter — you review it and send it yourself.

Because it captures the pane when you start, the text always lands in the agent you were talking to, even if focus wanders in the meantime.

🔒

Transcription uses Apple's on-device speech models (macOS 26's SpeechAnalyzer). Your audio never leaves your Mac. Dictation can be turned off completely in Settings.

17 Settings & opt-outs

mrd is designed to need no setup, but every automatic behavior can be turned off. Each one has a preference key and an environment variable; the preference is the easy way for everyday use.

To change a preference, run this in Terminal and then relaunch mrd:

# example: turn the notch HUD off
defaults write com.merdi.mrd mrd.NotchHUD -bool NO

# turn it back on
defaults write com.merdi.mrd mrd.NotchHUD -bool YES

18 Files & data

mrd keeps its own data in one place, and touches your agents' config only for hooks:

LocationWhat it is
~/Library/Application Support/mrd/Everything mrd owns: your saved layout, agent session records, scrollback, and its local control socket.
~/.claude/settings.jsonWhere mrd's Claude Code notification hooks live.
~/.codex/hooks.jsonWhere mrd's Codex notification hooks live.
/Applications/mrd.app/Contents/Resources/mrd-cliThe bundled command-line helper used by the hooks (and for manual hooks setup/remove).

Nothing here is required reading — it's listed so you know exactly what mrd writes and where, and what to remove if you ever uninstall.

19 Privacy

  • No telemetry. mrd doesn't track you or phone home about how you use it.
  • On-device speech. Dictation runs entirely on Apple's local speech models — your audio never leaves your Mac.
  • No screen scraping. Agent status comes from the agents' own hook events, not from reading your terminal output or keystrokes.
  • Local only. The one network thing mrd does on its own is check GitHub for a newer release — and you can switch even that off (Settings).

20 Troubleshooting

“Open Claude / Open Codex” isn't in the + menu

mrd only shows an agent if it can find that CLI on the same PATH your login shell uses. If you installed it in a way that isn't on your shell profile's PATH (e.g. a custom directory not exported in ~/.zshrc/~/.zprofile), add it there, then restart mrd. Test it by opening a normal Terminal and running which claude or which codex.

Codex isn't sending notifications / questions

Codex silently ignores hooks it hasn't been told to trust. Approve mrd's hooks once via Codex's hook-review prompt or its /hooks command. See Notifications & attention.

The microphone button isn't showing

The mic only appears when the focused pane is a Claude or Codex session — not for plain terminals. Also confirm dictation isn't disabled (mrd.Dictation), and that you granted Microphone and Speech Recognition access in System Settings → Privacy & Security.

macOS says the app is “damaged”

You're opening an old, un-notarized build. Download the newest release; current builds are notarized and open cleanly. To rescue an old copy: xattr -dr com.apple.quarantine /Applications/mrd.app.

Updates won't install

mrd can't replace itself if it's running from a read-only location (like a mounted disk image). Move mrd.app into Applications and try again.

An agent row is stuck on a status

Status is driven by agent hooks. If a row looks wrong, make sure hooks are installed and trusted (above). Reinstall them with mrd-cli hooks setup if needed. Keeping mrd updated also helps — status handling improves release to release.

21 Uninstall

  1. Quit mrd.
  2. Optionally remove its hooks first: /Applications/mrd.app/Contents/Resources/mrd-cli hooks remove
  3. Drag mrd.app from Applications to the Trash.
  4. Optionally delete its data: rm -rf ~/Library/Application\ Support/mrd

If you skip the hook-removal step, you can also delete mrd's entries by hand from ~/.claude/settings.json and ~/.codex/hooks.json.

Something missing or unclear? Open an issue on the releases repository.
